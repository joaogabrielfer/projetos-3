## 🌿 Fluxo de Versionamento

Para garantir a organização e evitar conflitos no código, a equipe utiliza um fluxo de versionamento baseado no **Git Flow Simplificado** e **Commits Semânticos**.

### Estrutura de Branches:
- `main`: Branch principal, contém apenas código testado, estável e pronto para produção (MVP). **Nenhum commit direto é permitido aqui.**
- `feature/`: Branches de desenvolvimento de novas funcionalidades (Histórias de Usuário). Ex: `feature/uh1-cadastro-usuario`.
- `bugfix/` ou `hotfix/`: Branches para correção de erros. Ex: `bugfix/erro-login`.

### Regras do Fluxo:
1. O desenvolvedor cria uma branch a partir da `main` (ex: `git checkout -b feature/uhX-nome-da-feature`).
2. Trabalha na funcionalidade fazendo commits semânticos:
   - `feat:` Para novas funcionalidades.
   - `fix:` Para correção de bugs.
   - `docs:` Para alterações na documentação.
   - `style:` Para formatação ou alterações de UI/CSS.
3. Após finalizar, faz o *push* e abre um **Pull Request (PR)** da sua branch para a `main`.
4. Outro membro da equipe faz o **Code Review** e aprova o PR.
5. Após o merge na `main`, a branch da feature pode ser deletada.
